构造函数

构造函数是一种特殊的成员函数，用于在创建对象时初始化对象的成员变量。与其他成员函数一样，构造函数位于类定义的内部，并且可以是公共的、私有的或受保护的。

只要类的对象被创建，就会执行构造函数。

C++中的构造函数有以下几个特点：

  1. 构造函数的名称必须与类名相同。
  2. 构造函数没有返回值，也不需要显式地声明返回类型。
  3. 构造函数可以有参数，这些参数可以用来传递初始化对象的值。
  4. 如果没有定义构造函数，编译器会生成默认构造函数。
  5. 如果定义了带参数的构造函数，则必须同时定义默认构造函数，否则会导致编译错误。

  1. 成员初始化列表中的初始化顺序与在类定义中声明的顺序无关，而是与在类定义中定义成员变量的顺序有关。成员初始化列表中应该按照类定义中的成员变量声明顺序来初始化成员变量。
  2. 成员初始化列表只能在构造函数中使用。
  3. const 和引用成员变量只能在成员初始化列表中初始化，不能在构造函数体中初始化。因为 const 和引用成员变量必须在创建对象时被初始化，并且不能被重新赋值。
  4. 必须初始化所有成员变量：使用成员初始化列表时，必须为所有成员变量进行初始化，否则会导致编译错误。
  5. 可以使用表达式和函数来初始化成员变量：成员初始化列表中的值可以是表达式或函数调用的返回值。

默认构造函数

如果一个类没有定义任何构造函数，编译器会自动为该类生成一个默认构造函数。默认构造函数是一个无参数的构造函数，它可以用来创建对象并初始化对象的成员变量。

如果一个类定义了带参数的构造函数，但没有定义默认构造函数，则该类只能通过显式调用带参数的构造函数来创建对象。

  1. 只能用于默认构造函数、拷贝构造函数、拷贝赋值函数和析构函数，不能用于其他类型的构造函数。
  2. 默认构造函数、拷贝构造函数、拷贝赋值函数和析构函数中的函数体必须为空。
  3. 如果类中定义了其他构造函数，那么必须在默认构造函数、拷贝构造函数和拷贝赋值函数中使用 = default 显式要求编译器生成函数。

委托构造函数

C++11引入委托构造函数，它允许一个构造函数调用另一个构造函数完成部分或全部的初始化工作。这个特性可以简化代码并提高代码重用性。

构造函数的委托必须在构造函数的初始化列表中完成，否则会导致编译错误。此外，一个类只能有一个委托构造函数，其他构造函数必须直接初始化对象的成员。

拷贝构造函数是在以下情况下自动调用的：

  • 通过另一个同类型对象初始化一个新对象时；
  • 将一个同类型对象作为参数传递给函数时；
  • 函数返回一个同类型对象时；
  • 从另一个同类型对象复制一个对象时。

如果没有显式定义拷贝构造函数，编译器会自动生成默认的拷贝构造函数，这个默认的拷贝构造函数会简单地将对象的成员变量复制到新对象中，如果类中有指针等动态分配的成员变量，可能会导致意想不到的错误，因此在需要时最好显式地定义拷贝构造函数。

初始化与赋值的区别：初始化发生在定义的时候，只能发生一次；赋值是在初始化完成后，可以发生多次。

转换构造函数

将其他类型转换为当前类的类型需要使用转换构造函数（conversion constructor）。

转换构造函数也是一种构造函数，它遵循构造函数的一般规则，允许将一种数据类型的对象隐式转换为另一种数据类型的对象。

在定义转换构造函数时，需要在构造函数前面添加关键字 explicit 或 implicit 来指定转换的类型，如果不指定则默认为 implicit。

转换构造函数只有一个参数。

过度使用转换构造函数可能会导致代码难以理解和维护，应该谨慎使用。

构造函数使用的关键字

explicit

关键字explicit可以用来声明构造函数或类型转换函数，表示该函数只能显式调用，不能隐式调用。

如果一个构造函数或类型转换函数被声明为explicit，那么它就不能用于隐式类型转换或隐式对象的创建。这样做的目的是为了避免程序中隐式转换导致的不明确行为和错误。如果一个函数不声明为explicit，那么它就可以被隐式调用。

constexpr

C++11 中引入了 constexpr 关键字，它用于声明一个可以在编译时求值的常量表达式，这样可以在编译期间对表达式进行求值，避免了在运行期间进行计算的开销。

构造函数也可以使用 constexpr 关键字来声明，表示该构造函数可以在编译期间被调用并求值。使用 constexpr 构造函数可以在编译期间生成常量对象，提高程序的效率。

在上面的示例在 main 函数中，使用 constexpr 构造函数创建了一个常量对象 obj，并使用 static_assert 进行断言，确保其值为 123。由于 constexpr 构造函数可以在编译期间进行求值，因此可以保证 obj 的值在编译期间就已经确定，不会受到运行时的影响。

需要注意的是，使用 constexpr 构造函数有一些限制:

  1. 必须对所有成员变量进行初始化;

  2. 初始化必须为常量表达式。
