异常处理机制用来捕获运行时的错误，让程序继续执行，或者在结束程序前保存内存中的数据、关闭文件、释放内存等。

抛出异常，就是报告一个运行时错误，程序根据错误信息进一步处理。

发生异常时必须将异常明确地抛出，try 才能检测到；如果不抛出来，即使有异常 try 也检测不到。抛出异常就是要明确地告诉程序放生了什么错误。

检测到异常后程序的执行会从异常点跳转到 catch 所在的位置，位于异常点之后的、并且在当前 try 块内的语句就都不会再执行。执行完 catch 块包含的语句后会继续执行后面的代码。

异常类型和多级 catch 匹配

异常类型可以是基本类型，也可以是聚合类型。C++本身和标准库函数抛出的异常，都是 exception 类或其子类的异常。

exceptionType variable和函数的形参类似，当异常发生后，会将异常数据传递给 variable 变量。

一个 try 后面可以跟多个 catch，当异常发生时，程序会按照从上到下的顺序，将异常类型和 catch 所能接收的类型逐个匹配。

catch 在匹配过程中会发生类型转换，只包括向上转型、const 转换和数组或函数指针转换，其他类型转换不能应用于catch。

抛出异常

throw关键字显示地抛出异常：

exceptionData 是“异常数据”的意思，可以包含任意的信息。exceptionData 可以是 int、float、bool 等基本类型，也可以是指针、数组、字符串、结构体、类等聚合类型。

throw关键字除了可以用在函数体中抛出异常，还可以用在函数头和函数体之间，指明当前函数能够抛出的异常类型，这称为异常规范（Exception specification），异常指示符或异常列表。

  • throw 指明该函数只能抛出 int 类型的异常，其他类型的异常 try 将无法捕获。
  • 如果函数抛出多种类型异常，可以用逗号隔开。
  • 如果函数不会抛出任何异常，()中什么也不写。

1. 虚函数的异常规范

派生类虚函数的异常规范必须与基类虚函数的异常规范一样或更严格。

2. 函数定义和声明的异常规范

异常规范在函数定义和函数声明中必须同时指明，并且必须一致。

异常规范在C++11中已经启用，不建议使用。

exception 类的直接派生类：

异常名称 说 明
logic_error 逻辑错误。
runtime_error 运行时错误。
bad_alloc 使用 new 或 new[ ] 分配内存失败时抛出的异常。
bad_typeid 使用 typeid 操作一个 NULL 指针，而且该指针是带有虚函数的类，这时抛出 bad_typeid 异常。
